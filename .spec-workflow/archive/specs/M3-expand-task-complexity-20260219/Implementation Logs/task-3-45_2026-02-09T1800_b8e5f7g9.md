### Components

#### ConnectionStatus (new)
- **Location:** client/src/components/ConnectionStatus.tsx
- **Purpose:** Persistent system connectivity indicator in the navigation header
- **Features:**
  - Compact dot indicator with overall status label (All Systems OK / Partial Connectivity / Systems Down)
  - Click-to-expand dropdown showing individual service statuses
  - WebSocket status: reactive via useWebSocket hook (connected/connecting/disconnected)
  - KĀDI Broker status: polled via /api/health every 30s
  - File Watcher status: polled via /api/health every 30s
  - Color-coded dots: green (connected/enabled), yellow (connecting/degraded), red (disconnected/down), gray (disabled/unknown)

  - Last-checked timestamp in dropdown footer

### Design Decisions

1. **Polling vs WebSocket for health** — Health endpoint is polled every 30s rather than pushed via WS because the health check itself validates the HTTP path independently of the WS connection.
2. **Overall status derivation** — healthy = WS + broker both connected; degraded = either one connected; down = both disconnected. File watcher is informational only (doesn't affect overall status).
3. **Compact + expandable** — Minimal footprint in the nav bar (just a dot + label), with full details on click.
